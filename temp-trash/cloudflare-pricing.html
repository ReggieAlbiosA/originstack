Cloudflare Pricing Guide

A comprehensive breakdown of Cloudflare's pricing for Workers, Pages, R2, D1, Images, and edge computing services.

Core Pricing Tiers

Workers Free

  • Cost: $0/month
  • Purpose: Personal projects, testing, and learning
  • Requests: 100,000 per day
  • CPU Time: 10ms per invocation
  • Use Case: Ideal for developers experimenting with edge computing, building hobby projects, or prototyping

Workers Paid (Standard)

  • Cost: $5/month minimum
  • Purpose: Production applications with significant traffic
  • Requests: 10 million/month included
  • CPU Time: 30 million CPU ms/month
  • Key Benefit: No bandwidth/egress charges
  • Use Case: Commercial websites, APIs, microservices

Enterprise

  • Cost: Custom pricing (contact sales)
  • Purpose: Large-scale applications with enterprise requirements
  • Features: Custom SLA, dedicated support, advanced security
  • Use Case: High-traffic applications, custom compliance needs

Cloudflare Workers

Requests

  • What it is: Number of times your Worker code is invoked by HTTP requests
  • Pricing:
    • Free: 100,000 requests per day (resets at midnight UTC)
    • Paid: 10 million requests/month + $0.30 per additional million
  • Important: Requests to static assets are FREE and UNLIMITED

CPU Time

  • What it is: Actual compute time your Worker uses during execution (measured in milliseconds)
  • Why it matters: Only active execution counts - waiting for APIs doesn't
  • Pricing:
    • Free: 10ms per invocation max
    • Paid: 30 million CPU ms/month + $0.02 per additional million
  • Maximum Limits:
    • HTTP requests: 5 minutes max (default: 30 seconds)
    • Cron Triggers: 15 minutes max
    • Queue Consumers: 15 minutes max

Duration

  • What it is: Total wall-clock time from start to end
  • Pricing: No charge or limit for duration
  • Example: Worker waits 10 seconds for API but uses only 50ms CPU = charged for 50ms only

Workers Static Assets

  • What it is: Host static files directly in your Worker
  • Pricing: Requests to static assets are FREE and UNLIMITED
  • Limits:
    • Free: 20,000 files per Worker version
    • Paid: 100,000 files per Worker version
    • Max file size: 25 MB per file

Durable Objects

What it is: Strongly consistent, stateful objects for coordination, real-time collaboration, and transactional storage at the edge.

Cloudflare Images

What it is: Image storage, optimization, and transformation service.

Free Plan

  • Images Transformed: 5,000 unique transformations/month (free)
  • Use Case: Optimize images stored outside Cloudflare (R2, external URLs)

Paid Plan

  • Images Transformed: 5,000 included + $0.50 per 1,000 unique transformations/month
  • Images Stored: $5 per 100,000 images/month
  • Images Delivered: $1 per 100,000 images delivered/month

Transformations Details

  • Unique Transformation: Counted once per 30-day sliding window
  • Format Auto: Counts as 1 transformation (not multiple for AVIF/WebP)
  • Supported Formats: JPEG, PNG, GIF, WebP, AVIF, SVG, HEIC

Cost Optimization Tips

  1. Leverage Static Assets: Serve static content directly to avoid Worker invocations
  2. Optimize CPU Time: Minimize computationally expensive operations
  3. Cache Aggressively: Use Cache API or KV to reduce repeated processing
  4. Batch Operations: Group multiple operations to reduce request counts
  5. Use Service Bindings: Inter-Worker communication is free on Standard pricing
  6. Monitor Limits: Set CPU limits to prevent runaway costs
  7. Smart KV Usage: Use KV for read-heavy workloads
  8. Durable Objects: Let objects hibernate when inactive
  9. Use Indexes: D1 indexes reduce rows read significantly
  10. R2 for Media: Store large files in R2 with zero egress fees

Frequently Asked Questions

Do I get charged for failed requests?

No. You are not charged for requests that return unauthorized (401) errors.

What happens if I exceed Free plan limits?

Most resources will stop working until the daily reset at midnight UTC. Upgrade to Paid for production.

Are there egress/bandwidth charges?

No! Cloudflare does not charge for data transfer, egress, or bandwidth on Workers, R2, D1, or KV.

How is CPU time calculated?

Only active execution counts. Waiting for external APIs or database queries doesn't count toward CPU time.

Can I use multiple storage products together?

Yes! Use R2 for large files, KV for read-heavy data, D1 for relational data, and Durable Objects for stateful coordination.

Do static asset requests count toward Worker requests?

No! Requests to static assets are FREE and UNLIMITED on all plans.
