Vercel Pricing Guide

A comprehensive breakdown of Vercel's pricing components and their role in modern web hosting.

Core Pricing Tiers

Hobby (Free)

  • Cost: Free forever
  • Purpose: Personal projects and experimentation
  • Use Case: Perfect for developers learning, building portfolios, or testing applications without financial commitment
  • Team Size: 1 user (personal use only)

Pro ($20/user/month)

  • Cost: $20 per user/month, plus usage-based pricing
  • Purpose: Professional developers and small teams
  • Use Case: Production applications with moderate to high traffic, freelance projects, and small business websites
  • Team Size: Unlimited team members ($20/additional member) with free viewer seats
  • Included Credit: $20 of usage credit per month

Enterprise (Custom Pricing)

  • Cost: Custom pricing based on requirements
  • Purpose: Large organizations with specific needs
  • Use Case: High-traffic applications, enterprise-grade SLA requirements, dedicated support, and advanced security features
  • Features: 99.99% SLA, advanced support, custom contracts

Compute & Functions

Fluid Compute (New Model - 2024+)

  • What it is: Modern compute model replacing traditional GB-Hours calculation with Active CPU time and Provisioned Memory
  • Why it matters: More accurate billing based on actual CPU usage rather than memory-time combinations
  • Pricing Impact:
    • Active CPU Time:
      • Hobby: 4 hours/month included
      • Pro: Included in usage credits, then $0.128 per hour
    • Provisioned Memory:
      • Hobby: 360 GB-hours/month included
      • Pro: Included in usage credits, then $0.0106 per GB-hour
  • Hosting Context: Handles API routes, authentication, database queries, and dynamic content generation

Serverless Function Invocations

  • What it is: Number of times functions are called
  • Why it matters: Each API request, form submission, or dynamic page load may trigger function execution
  • Pricing Impact:
    • Hobby: 100,000 invocations/month
    • Pro: Unlimited invocations (no cap)
  • Hosting Context: High-traffic sites need unlimited invocations to handle peak loads

Function Duration Limits

  • What it is: Maximum execution time per function (configurable via maxDuration)
  • Why it matters: Determines how long a function can run before timing out
  • Limits:
    • Hobby: 10 seconds max (default)
    • Pro: 15 seconds default, configurable up to 60 seconds (or 300s with special config)
    • Enterprise: Configurable based on needs
  • Configuration: Set via export const maxDuration = 15; in code or vercel.json
  • Hosting Context: Longer durations needed for complex operations like report generation, video processing, or external API calls

Edge Network & CDN

Fast Data Transfer

  • What it is: The amount of data transferred from Vercel's edge network to end users
  • Why it matters: Every time someone visits your website, images load, or APIs are called, data is transferred. Vercel's global CDN ensures fast delivery worldwide
  • Pricing Impact:
    • Hobby: 100 GB/month included
    • Pro: 1 TB/month included
    • Additional: $0.15 per GB
  • Hosting Context: High bandwidth is crucial for media-rich sites, popular blogs, e-commerce platforms, and any high-traffic application

Edge Requests

  • What it is: Number of requests handled at the edge (CDN level)
  • Why it matters: Edge requests are served from the nearest geographic location to your users, providing ultra-low latency
  • Pricing Impact:
    • Hobby: 1 million requests/month
    • Pro: 10 million requests/month
    • Additional: Usage-based pricing
  • Hosting Context: Essential for global performance - users in Asia, Europe, and Americas all get fast response times

Fast Origin Transfer

  • What it is: Data transferred from your origin (source) to Vercel's edge network
  • Why it matters: First-time requests fetch data from origin and cache at edge
  • Pricing Impact:
    • Hobby: 10 GB/month included
    • Pro: Included in usage credits
    • Additional: $0.40 per GB
  • Hosting Context: Important for dynamic content and cache misses

Build & Deployment

Build Execution Time

  • What it is: Time spent compiling and building your application
  • Why it matters: Every deployment triggers a build process (compiling TypeScript, bundling assets, generating static pages)
  • Pricing Impact:
    • Hobby: No specific limit (fair use)
    • Pro: 200 hours/month included, then usage-based
    • Additional: Usage-based pricing after included amount
  • Hosting Context: Large applications or frequent deployments consume more build time; essential for CI/CD workflows

Build Performance

  • What it is: Build speed and queue priority
  • Benefits:
    • Hobby: Standard build speed, may queue during high usage
    • Pro: Faster builds with no queues, priority processing
    • Enterprise: Custom build infrastructure
  • Hosting Context: Teams need fast, reliable builds to maintain development velocity

Concurrent Builds

  • What it is: Number of builds that can run simultaneously
  • Why it matters: Multiple team members pushing code or preview deployments can trigger parallel builds
  • Limits: Based on plan tier and team size
  • Hosting Context: Teams need concurrent builds to avoid deployment queues

Web Analytics & Monitoring

Web Analytics Events

  • What it is: Built-in privacy-friendly analytics (no cookies required)
  • Why it matters: Track page views, user behavior, and site performance without third-party tools
  • Pricing Impact:
    • Hobby: 50,000 events/month included
    • Pro: Included in base, then $3 per 100,000 events (prorated)
    • Enterprise: Custom pricing
  • Features:
    • 1-month reporting window (Hobby)
    • Extended reporting (Pro/Enterprise)
    • Privacy-compliant (GDPR-friendly)
    • No consent banners required
  • Hosting Context: Essential for understanding user behavior and site performance

Web Analytics Data Transfer

  • What it is: Bandwidth used by analytics collection
  • Pricing Impact:
    • Hobby: 10 GB/month included
    • Pro: Included in bandwidth allocation
  • Hosting Context: Analytics data collection has minimal bandwidth impact

Real User Monitoring (Speed Insights)

  • What it is: Track actual user performance metrics (Core Web Vitals, etc.)
  • Why it matters: Monitor real-world performance, not just synthetic tests
  • Pricing Impact: Included in Pro and Enterprise
  • Hosting Context: Essential for maintaining SEO rankings and user experience quality

Storage Products

Vercel Blob Storage

  • What it is: Object storage for files, images, and media
  • Why it matters: Store and serve large files without bundling in deployments
  • Pricing Impact:
    • Storage Size: 1 GB included, then $0.023 per GB/month
    • Simple Operations: 10,000 included, then $0.40 per 1M
    • Advanced Operations: 2,000 included, then $5.00 per 1M
    • Data Transfer: 10 GB included, then $0.05 per GB
    • Edge Requests: 1M/month included
  • Hosting Context: Essential for user uploads, media libraries, and asset management

Vercel KV (Redis)

  • What it is: Redis-compatible key-value database at the edge
  • Why it matters: Fast data access for caching, sessions, and simple data storage
  • Pricing Impact: Separate usage-based pricing
  • Use Cases: Rate limiting, feature flags, shopping carts, session management

Vercel Postgres

  • What it is: Serverless Postgres database
  • Why it matters: Full relational database without infrastructure management
  • Pricing Impact: Separate usage-based pricing
  • Use Cases: Structured data, user accounts, complex queries

Developer Tools & Features

Vercel Sandbox

  • What it is: Cloud development environments for testing and development
  • Why it matters: Spin up isolated environments for experimentation
  • Pricing Impact:
    • Active CPU: 5 hours/month included, then $0.128/hour
    • Provisioned Memory: 420 GB-hours/month included, then $0.0106/GB-hour
    • Network: 20 GB/month included, then $0.15/GB
    • Sandbox Creations: 5,000/month included, then $0.60 per 1M
    • Concurrent Sandboxes: 10 (Hobby)
  • Hosting Context: Rapid prototyping and testing without affecting production

Edge Config

  • What it is: Ultra-fast, globally replicated configuration store with sub-millisecond read access
  • Why it matters: Store feature flags, redirects, and config data that needs instant global availability
  • Pricing Impact:
    • Reads: 1 million/month included (all plans)
    • Writes: 100/month included (all plans)
    • Additional usage: Contact sales for higher limits
  • Access Methods: SDK (@vercel/edge-config), REST API with read tokens
  • Use Cases: Feature flags, A/B testing, dynamic configuration, LaunchDarkly/Statsig integration

Image Optimization

  • What it is: Automatic image resizing, format conversion (WebP/AVIF), and compression via Next.js <Image> component
  • Why it matters: Images are often the largest assets on websites; optimization reduces load times and bandwidth
  • Pricing Impact:
    • Cache Reads: 300,000/month included (all plans)
    • Cache Writes: 100,000/month included (all plans)
    • Additional usage: Usage-based pricing for overages
  • Optimization Features:
    • Automatic format selection (WebP, AVIF)
    • Responsive image sizing
    • Lazy loading
    • Remote and local image support
  • Cost Control: Use unoptimized prop for small/SVG/animated images, configure minimumCacheTTL for longer caching
  • Hosting Context: Critical for e-commerce, blogs, portfolios, and any image-heavy site

Cost Optimization Strategies

  1. Maximize Edge Caching

    • Use Cache-Control headers effectively
    • Leverage Vercel's CDN for static assets
    • Implement ISR for dynamic content with reasonable revalidation times

  2. Optimize Compute Usage

    • Minimize function execution time
    • Use edge functions for simple logic (cheaper than serverless)
    • Implement efficient database queries
    • Cache expensive operations

  3. Image Optimization

    • Always use Next.js <Image> component or @vercel/og
    • Serve images in modern formats (WebP/AVIF)
    • Use appropriate image sizes
    • Leverage image CDN caching

  4. Static Generation Strategy

    • Pre-render pages at build time (SSG) when possible
    • Use ISR for content that updates periodically
    • Reserve SSR for truly dynamic content

  5. Monitor and Alert

    • Set up spend management alerts
    • Track usage in Vercel dashboard
    • Identify usage spikes and optimize
    • Right-size your plan based on actual usage

  6. Efficient Data Transfer

    • Compress API responses
    • Minimize payload sizes
    • Use pagination for large datasets
    • Implement efficient data fetching strategies

  7. Smart Architecture

    • Use edge functions for authentication checks
    • Implement rate limiting to prevent abuse
    • Cache API responses when appropriate
    • Use Vercel KV for session storage instead of database

Frequently Asked Questions

Do I get charged for preview deployments?

Preview deployments consume bandwidth, build time, and compute resources just like production, but they're included in your plan limits.

What happens if I exceed my limits on Hobby?

Most resources will stop working until the monthly reset. Consider upgrading to Pro for production applications.

Can I set spending limits?

Yes, Pro and Enterprise plans offer spend management with configurable limits and alerts.

Are serverless functions always running?

No, they spin up on-demand (cold start), but Pro includes cold start prevention for critical functions.

How does the $20 usage credit work?

On Pro plan, each member includes $20 of usage credit that applies to overages before additional charges.

Is bandwidth charged for failed requests?

No, only successful responses consume bandwidth allocation.

Important Clarifications

  1. Cron Jobs: Hobby plans have hourly accuracy (* 8 * * *), while Pro+ have minute-level accuracy (5 8 * * *)
  2. Routing Middleware: Runs on Edge runtime by default, can be configured to use Node.js runtime
  3. Build Cache: Automatically enabled for all frameworks (Next.js: .next/cache/**, Nuxt: .nuxt/**, etc.)
  4. waitUntil(): Available for extending request lifetime for background tasks after response is sent
  5. SIGTERM Handling: 500ms cleanup window for graceful function shutdown
  6. Image Optimization Paths: /_next/image (Next.js) or /_vercel/image (general)
  7. Multipart Upload: Supported for Vercel Blob with minimum 5MB per part (except last part)
  8. Edge Config Updates: Instant global propagation (sub-millisecond reads) after write
  9. Analytics Privacy: GDPR-compliant, no cookies required, can implement beforeSend filtering
  10. Function Regions: Configurable via regions in vercel.json or preferredRegion in code
